# flowservice — Implementation TODO

Rust service that replaces `apps/searchservice` (Go) and `apps/eventsservice` (Go).

**What it does:**
- Serves `SearchService` gRPC → queries Meilisearch
- Serves `NotificationService` gRPC → queries PostgreSQL `notifications` table
- Consumes `activity` Kafka topic → writes/deletes notification rows and fan-out feed rows
- Consumes `entity-changes` Kafka topic → deletes stale notification rows and fan-out feed rows on post create/delete
- Runs a periodic cleanup task → deletes old `outbox` and `feed` rows

**Before starting:** Read `docs/asyncservice-proposal.md` for design rationale and Rust
pattern examples. Read `apps/authservice/src/` — every pattern here mirrors it.

## Step 4 — search/meili.rs

This ports `apps/searchservice/search/meili.go` to Rust.

**What it must do:**
1. On startup: connect with the master key, check whether a scoped key with UID
   `"flowservice-scoped-key"` already exists (idempotent). If not, create it with
   actions `["search", "indexes.get"]` and indexes `["users", "posts", "hashtags"]`.
   **Migration note:** the Go searchservice used UID `"searchservice-scoped-key"`. On
   first deploy of flowservice a new key is created; the old one is orphaned in
   Meilisearch but harmless. No manual cleanup is required.
2. On startup: ensure the three indexes exist with the correct searchable/filterable/
   sortable attribute settings (see Go source for the exact settings per index).
3. Return a `MeiliClient` that holds the scoped-key client. The master key must not be
   retained after startup.
4. Expose `search_users`, `search_posts`, `search_hashtags` methods — each calls the
   underlying `meilisearch-sdk` index search and returns typed results.

**Cursor encoding** (port from `apps/searchservice/search/pagination.go`): the cursor
is base64url-encoded JSON `{"offset": N}`. Clamp offset to `[0, 1000]`.

**Hit mapping** (port from `apps/searchservice/search/controller.go`):
- `id` field arrives as `f64` from JSON — cast to `i32`
- User hit → `{id, username, name}`
- Post hit → `{id, content, created_at → created}`
- Hashtag hit → `{id, name, post_count}`

**`MeiliClient` must derive `Clone`** — `SearchController` holds it and tonic requires
controllers to be `Clone` (enforced at compile time by `add_service`). Verify that the
`meilisearch-sdk` `Client` type is `Clone`; if not, wrap it in `Arc`.

**Important:** Use `reqwest` (already a transitive dep of `meilisearch-sdk`) for any
direct HTTP calls to the Meili key management API if the SDK does not expose it.

**Define a plain data struct** (not a proto type):

```rust
pub struct Notification {
    pub id: i64,
    pub external_id: i64,
    pub user_id: i32,
    pub actor_id: i32,
    pub notif_type: String,
    pub entity_id: String,
    pub read: bool,
    pub created: chrono::DateTime<chrono::Utc>,

```

**Implement the trait on `sqlx::PgPool`.** SQL to port (use `sqlx::query_as` not

builds; see the comment in `apps/authservice/src/db_client.rs:46`):

- `list`: cursor-based pagination on `(created DESC, id DESC)` — decode cursor as

- `unread_count`: `SELECT COUNT(*)::int FROM notifications WHERE user_id = $1 AND read = false`
- `delete_by_entity`: `DELETE FROM notifications WHERE type = ANY($1) AND entity_id = $2`
- `delete_by_actor_and_type`: `DELETE FROM notifications WHERE actor_id=$1 AND user_id=$2 AND type=$3 AND entity_id=$4`

## Step 8 — notifications/consumer.rs

Port `apps/eventsservice/internal/notifications/consumer.go`.

**Kafka topics to subscribe:** `"activity"` and `"entity-changes"`.

**Event types** — use serde tagged enums instead of the Go map dispatch:

```rust
#[derive(serde::Deserialize)]
#[serde(tag = "op", rename_all = "snake_case")]
enum ActivityEvent {
    Like     { _outbox_id: i64, actor_id: i32, recipient_id: i32, post_id: i64 },
    Unlike   { actor_id: i32, recipient_id: i32, post_id: i64 },
    Repost   { _outbox_id: i64, actor_id: i32, recipient_id: i32, post_id: i64 },
    Unrepost { actor_id: i32, recipient_id: i32, post_id: i64 },
    Reply    { _outbox_id: i64, actor_id: i32, recipient_id: i32, reply_post_id: i64 },
    Unreply  { reply_post_id: i64 },
    Follow   { _outbox_id: i64, actor_id: i32, recipient_id: i32 },
    Unfollow { actor_id: i32, recipient_id: i32 },
    #[serde(other)]
    Unknown,
}

#[derive(serde::Deserialize)]
struct EntityChangeEvent {
    table: String,
    op: String,
    id: Option<i64>,
}
```

**Self-notification guard:** skip insert when `actor_id == recipient_id` (the Go
code has this check — see `notifications/consumer.go:157,185,199`).

**Before the loop, subscribe to topics** — rdkafka's `subscribe` replaces franz-go's
`AddConsumeTopics`. Do this once before entering the poll loop:
```rust
consumer.subscribe(&["activity", "entity-changes"])
    .expect("kafka topic subscription failed");
```

**Consumer function signature** — takes a `tokio::sync::watch::Receiver<bool>` cloned
from main.rs (`.changed()` is the watch API; this matches authservice's cleanup task):
```rust
pub async fn run(
    consumer: rdkafka::consumer::StreamConsumer,
    db: sqlx::PgPool,
    mut shutdown_rx: tokio::sync::watch::Receiver<bool>,
) { ... }
```

**Consumer loop pattern** (at-least-once; matches the Go implementation):

```rust
loop {
    tokio::select! {
        _ = shutdown_rx.changed() => break,
        result = consumer.recv() => match result {
            Err(e) => tracing::warn!(error = %e, "kafka recv error"),
            Ok(msg) => {
                let payload = msg.payload().unwrap_or_default();
                if let Err(e) = dispatch(&db, msg.topic(), payload).await {
                    tracing::warn!(
                        topic = msg.topic(), offset = msg.offset(),
                        error = %e, "notification event failed"
                    );
                    // Do NOT commit — let the message replay (at-least-once)
                    break;
                }
                consumer.commit_message(&msg, rdkafka::consumer::CommitMode::Async).ok();
            }
        }
    }
}
```

The consumer task restarts on break — in `main.rs`, wrap it in a retry loop or accept
that the pod will be restarted by Kubernetes on consumer group failure.

**Dispatch logic** (port from `notifications/consumer.go:71-148`):

- Topic `"entity-changes"`: parse as `EntityChangeEvent`; if `table == "posts"` and
  `op == "delete"`, call `db.delete_by_entity(post_id_as_str, &["like","repost","reply"])`.
- Topic `"activity"`: deserialize as `ActivityEvent` via serde; match on variant:
  - `Like` → `db.insert(outbox_id, recipient_id, actor_id, "like", post_id_str)` (skip if actor==recipient)
  - `Unlike` → `db.delete_by_actor_and_type(actor_id, recipient_id, "like", post_id_str)`
  - `Repost` → `db.insert(outbox_id, recipient_id, actor_id, "repost", post_id_str)` (skip if actor==recipient)
  - `Unrepost` → `db.delete_by_actor_and_type(actor_id, recipient_id, "repost", post_id_str)`
  - `Reply` → `db.insert(outbox_id, recipient_id, actor_id, "reply", reply_post_id_str)` (skip if actor==recipient)
  - `Unreply` → `db.delete_by_entity(reply_post_id_str, &["reply"])`
  - `Follow` → `db.insert(outbox_id, recipient_id, actor_id, "follow", actor_id_str)` (skip if actor==recipient)
  - `Unfollow` → `db.delete_by_actor_and_type(actor_id, recipient_id, "follow", actor_id_str)`
  - `Unknown` → log and skip

## Step 10 — feed/db.rs

Port `apps/eventsservice/internal/feed/store/postgres/store.go`.

**Trait:**

```rust
#[async_trait::async_trait]
pub trait FeedDb: Send + Sync + 'static {
    // Bulk insert feed rows; idempotent — table has UNIQUE(user_id, post_id).
    // Port Go's bulkInsertBatch with ON CONFLICT (user_id, post_id) DO NOTHING.
    // Batch at most 1000 rows per statement (const bulkInsertBatchSize = 1000).
    async fn bulk_insert(&self, entries: &[Entry]) -> Result<(), sqlx::Error>;

    async fn prune_by_followee(
        &self, follower_id: i32, followee_id: i32,
    ) -> Result<(), sqlx::Error>;

    async fn count_followers(&self, author_id: i32) -> Result<i32, sqlx::Error>;

    async fn get_followers(&self, author_id: i32) -> Result<Vec<i32>, sqlx::Error>;

    // Last N non-reply posts by user_id, ordered by (created DESC, id DESC).
    async fn get_last_posts(&self, user_id: i32, limit: i32) -> Result<Vec<Entry>, sqlx::Error>;

    async fn get_fan_out_disabled(&self, user_id: i32) -> Result<bool, sqlx::Error>;
    async fn set_fan_out_disabled(&self, user_id: i32) -> Result<(), sqlx::Error>;

    // Periodic cleanup — called from the cleanup task.
    async fn delete_old_feed(&self) -> Result<(), sqlx::Error>;
    async fn delete_old_outbox(&self) -> Result<(), sqlx::Error>;
}

pub struct Entry {
    pub user_id: i32,
    pub post_id: i32,
    pub created: chrono::DateTime<chrono::Utc>,
}
```

**SQL to port:**

- `bulk_insert`: build a multi-row `INSERT INTO feed (user_id, post_id, created) VALUES ($1,$2,$3),($4,$5,$6),...  ON CONFLICT (user_id, post_id) DO NOTHING`. Use `$N` placeholders (sqlx does not support array binding for multi-row inserts without a loop or `UNNEST`). Alternatively: `INSERT INTO feed (user_id, post_id, created) SELECT * FROM UNNEST($1::int4[], $2::int4[], $3::timestamptz[]) ON CONFLICT (user_id, post_id) DO NOTHING` — simpler and avoids dynamic SQL.
- `prune_by_followee`: `DELETE FROM feed f USING posts p WHERE f.post_id = p.id AND f.user_id = $1 AND p.user_id = $2`
- `count_followers`: `SELECT COUNT(*)::int FROM followers WHERE user_id = $1`
- `get_followers`: `SELECT follower_id FROM followers WHERE user_id = $1`
- `get_last_posts`: `SELECT id, created FROM posts WHERE user_id = $1 AND in_reply_to_id IS NULL ORDER BY created DESC, id DESC LIMIT $2`
- `get_fan_out_disabled`: `SELECT fan_out_disabled FROM users WHERE id = $1`
- `set_fan_out_disabled`: `UPDATE users SET fan_out_disabled = true WHERE id = $1`
- `delete_old_feed`: `DELETE FROM feed WHERE created < now() - interval '30 days'`
- `delete_old_outbox`: `DELETE FROM outbox WHERE created < now() - interval '7 days'`

## Step 12 — src/main.rs

This wires everything together. The structure mirrors `apps/authservice/src/main.rs`
but with multiple spawned tasks.

```rust
#[allow(clippy::all)]
#[rustfmt::skip]
pub mod cogito;

mod internal_auth;
mod logging;
mod search;
mod notifications;
mod feed;

// ... use imports ...

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    logging::init();

    // --- Env vars ---
    let port              = std::env::var("PORT").unwrap_or_else(|_| "5050".to_string());
    let db_url            = std::env::var("DATABASE_URL").expect("DATABASE_URL required");
    let kafka_brokers     = std::env::var("REDPANDA_BROKERS").unwrap_or_else(|_| "localhost:9092".to_string());
    let valkey_url        = std::env::var("VALKEY_URL").unwrap_or_else(|_| "redis://localhost:6379".to_string());
    let meili_host        = std::env::var("MEILI_HOST").expect("MEILI_HOST required");
    let meili_master_key  = std::env::var("MEILI_MASTER_KEY").expect("MEILI_MASTER_KEY required");
    let fan_out_threshold = std::env::var("FAN_OUT_THRESHOLD")
        .ok().and_then(|v| v.parse().ok()).unwrap_or(10_000i32);

    let addr = format!("0.0.0.0:{}", port).parse()?;

    // --- Shared DB pool ---
    let pool = sqlx::postgres::PgPoolOptions::new()
        .max_connections(10)
        .connect(&db_url)
        .await?;

    // --- Meilisearch: provision key + indexes before accepting traffic ---
    let meili = search::meili::MeiliClient::new(&meili_host, &meili_master_key).await?;

    // --- Valkey / fred ---
    // Note: verify fred v9 API when implementing. The constructor signature and
    // from_url method name may differ slightly between minor versions.
    // fred v9 docs: https://docs.rs/fred/latest/fred/
    let valkey_config = fred::prelude::RedisConfig::from_url(&valkey_url)?;
    let valkey_client = fred::prelude::RedisClient::new(valkey_config, None, None, None);
    valkey_client.connect();
    valkey_client.wait_for_connect().await?;
    let follower_cache = feed::cache::FollowerCache::new(valkey_client);

    // --- Kafka consumers (two separate consumer groups) ---
    let notif_consumer = build_kafka_consumer(&kafka_brokers, "flowservice-notifications")?;
    let feed_consumer  = build_kafka_consumer(&kafka_brokers, "flowservice-feed")?;

    // --- Shutdown channel (watch, same pattern as authservice/src/main.rs) ---
    // watch::Receiver is Clone; each task gets its own clone and waits on .changed().
    let (shutdown_tx, shutdown_rx) = tokio::sync::watch::channel(false);

    // --- Spawn background tasks ---
    {
        let db = pool.clone();
        let rx = shutdown_rx.clone();
        tokio::spawn(async move {
            notifications::consumer::run(notif_consumer, db, rx).await;
        });
    }
    {
        let db = pool.clone();
        let cache = follower_cache;
        let rx = shutdown_rx.clone();
        tokio::spawn(async move {
            feed::consumer::run(feed_consumer, db, cache, fan_out_threshold, rx).await;
        });
    }
    {
        let db = pool.clone();
        let rx = shutdown_rx.clone();
        tokio::spawn(async move {
            run_cleanup(db, rx).await;
        });
    }

    // --- gRPC server ---
    let auth = internal_auth::interceptor;

    let search_ctrl = search::controller::SearchController::new(meili);
    let notif_ctrl  = notifications::controller::NotificationController::new(pool.clone());

    let shutdown_signal = async move {
        tokio::signal::ctrl_c().await.expect("ctrl_c handler failed");
        shutdown_tx.send(true).ok();
        tracing::info!("shutting down");
    };

    tracing::info!(port = %port, "server starting");

    tonic::transport::Server::builder()
        .add_service(cogito::search_service_server::SearchServiceServer::with_interceptor(
            search_ctrl, auth,
        ))
        .add_service(cogito::notification_service_server::NotificationServiceServer::with_interceptor(
            notif_ctrl, auth,
        ))
        .serve_with_shutdown(addr, shutdown_signal)
        .await?;

    pool.close().await;
    Ok(())
}

async fn run_cleanup(pool: sqlx::PgPool, mut shutdown_rx: tokio::sync::watch::Receiver<bool>) {
    let mut interval = tokio::time::interval(std::time::Duration::from_secs(3600));
    loop {
        tokio::select! {
            _ = interval.tick() => {
                if let Err(e) = sqlx::query("DELETE FROM outbox WHERE created < now() - interval '7 days'")
                    .execute(&pool).await
                {
                    tracing::warn!(error = %e, "outbox cleanup failed");
                }
                if let Err(e) = sqlx::query("DELETE FROM feed WHERE created < now() - interval '30 days'")
                    .execute(&pool).await
                {
                    tracing::warn!(error = %e, "feed cleanup failed");
                }
            }
            _ = shutdown_rx.recv() => {
                tracing::info!("cleanup task shutting down");
                break;
            }
        }
    }
}

fn build_kafka_consumer(
    brokers: &str,
    group_id: &str,
) -> Result<rdkafka::consumer::StreamConsumer, rdkafka::error::KafkaError> {
    rdkafka::ClientConfig::new()
        .set("bootstrap.servers", brokers)
        .set("group.id", group_id)
        .set("enable.auto.commit", "false")
        .set("auto.offset.reset", "earliest")
        .create()
}
```
